# 跨性别群体声音收集分析系统 - 项目计划书

## 一、项目概述

### 1.1 项目背景
本项目旨在为跨性别群体提供一个便捷的声音分析与记录工具，通过 Telegram Bot 作为交互界面，用户可以随时上传语音并获得详细的声音参数分析报告。

### 1.2 核心功能
- **语音上传与分析**：用户通过 Telegram 发送语音消息，系统自动下载并分析
- **分级信息展示**：`/analyze` 简洁模式 + `/expert` 专业模式，按需获取深度信息
- **多维度音频参数提取**：基频、共振峰、频谱、能量、音质等多类参数
- **可视化图表生成**：语谱图、基频轮廓图、共振峰轨迹图等（按需生成）
- **历史数据追踪**：记录历次录音及分析结果，支持对比，无数量上限
- **性别特征评估**：基于多组统计特征综合评估声音性别属性

### 1.3 运行平台
- 目标环境：Android Termux (Python 环境)
- 通信接口：Telegram Bot API
- 无需传统前端，纯命令交互

### 1.4 音频处理限制
- **单次音频最大时长**：120 秒（2 分钟）
- **超时长处理**：Bot 会提示音频过长，拒绝分析并引导用户重新录制
- **自动裁剪**：静音段自动移除，只分析有效语音段
- **默认采样率**：32000 Hz（32kHz），可在设置中调整

### 1.5 设置系统
- **入口命令**：`/settings`
- **交互方式**：内联按钮菜单，分层导航
- **可调参数**：采样率、基频算法、性别评估权重、图表样式等
- **持久化存储**：用户设置保存在数据库，跨会话生效

### 3.1 Telegram Bot 交互模块

#### 3.1.1 命令系统

| 命令 | 功能 | 说明 |
|------|------|------|
| `/start` | 启动机器人，显示欢迎和帮助 | 首次使用引导 |

#### 3.1.2 交互流程

**简洁分析流程 (`/analyze`)**
```
1. 用户发送语音消息
2. Bot 自动接收并暂存
3. 用户回复该语音并发送 /analyze
4. Bot 检查时长（≤120秒）
5. 执行完整参数提取（后台全量计算）
6. 返回核心参数文字报告（不包含图表）
7. 分析数据存入数据库
```

**专业分析流程 (`/expert`)**
```
1. 用户对已有分析记录发送 /expert
   （或发送 /expert <id> 指定记录）
2. Bot 返回内联按钮菜单：
   ┌─────────────────────────────┐
   │  📊 选择专业分析项目          │
   ├─────────────────────────────┤
   │  🎵 基频详细分析             │
   │  🎤 共振峰详细分析           │
   │  🌈 频谱特征分析             │
   │  🔊 能量与响度分析           │
   │  ✨ 音质与稳定性分析         │
   │  🗣️ 发声模式分析             │
   │  📈 语言节奏与语调           │
   │  🧭 性别特征深度分析         │
   │  🖼️  可视化图表（全部）       │
   └─────────────────────────────┘
3. 用户点击按钮后，Bot 返回对应详细信息 + 图表
4. 图表按需生成并缓存，重复查看直接从缓存取
```

#### 3.1.3 内联按钮菜单设计

```

  基频详细分析    →  F0范围/Jitter/音高轮廓图

  性别特征深度分析 →  多维度性别指数/分布区间
  可视化图表(全部) →  一次性发送所有图表
```

### 3.2 音频分析引擎模块

#### 3.2.1 核心依赖库
```python

python-telegram-bot   # Telegram Bot API
webrtcvad             # 语音端点检测 (可选)
```

#### 3.2.2 基础音频预处理
- 格式转换 (ogg/opus/mp3 → wav 单声道)
- 重采样 (默认 32kHz，可在设置中调整：16k/22.05k/32k/44.1k)
- 音量归一化（可开关）
- 静音检测与去除 (VAD，灵敏度可调)
- 时长校验（默认 ≤ 120 秒，可在设置中调整）

#### 3.2.3 音频参数提取模块

> **设计原则**：`/analyze` 命令触发时，后台执行**全部参数提取**并存入数据库。
> 用户通过 `/expert` 按需查看时，直接从缓存读取，避免重复计算。

**【1】基频与音高分析**

| 参数 | 说明 | 算法/实现 | 简洁模式 |
|------|------|-----------|:--------:|
| 平均基频 (F0 Mean) | 声带振动平均频率 | librosa.pyin / crepe | ✅ |

| 基频范围 (F0 Range) | 最小/最大基频 | min/max of valid_f0 | ✅ |

| 基频变异系数 | 归一化的音高波动 | std / mean | - |
| Jitter (local) | 相邻周期基频变化率 | 周期差异计算 | - |
| Jitter (rap) | 相对平均扰动 | 三点平均法 | - |
| 音高轮廓 (Pitch Contour) | 句子层面音高曲线 | 平滑插值序列 | - |

### 3.2.4 性别特征评估算法设计

**评估方法：多组统计特征 + 加权评分**

```
输入: 提取的全部音频参数
输出: 各维度性别指数 + 综合评分

步骤:
  1. 参数归一化
     - 每个参数映射到 [0, 1] 区间
     - 0 = 完全男性典型值
     - 1 = 完全女性典型值
     - 映射函数基于语音学文献的统计数据

  2. 维度独立评分
     基频维度 (权重最高):
       - F0 均值 → g_f0
       - F0 范围 → g_f0_range
       - F0 变异系数 → g_f0_var
       基频指数 = 0.7*g_f0 + 0.2*g_f0_range + 0.1*g_f0_var

     共振峰维度:
       - F1 → g_f1
       - F2 → g_f2
       - F3 → g_f3
       - F2-F1 间距 → g_f2f1
       共振峰指数 = 0.35*g_f1 + 0.35*g_f2 + 0.15*g_f3 + 0.15*g_f2f1

     音色维度:
       - 频谱质心 → g_centroid
       - 频谱滚降 → g_rolloff
       - 频谱带宽 → g_bandwidth
       音色指数 = 0.5*g_centroid + 0.3*g_rolloff + 0.2*g_bandwidth

     韵律维度:
       - 语调范围 → g_int_range
       - 语速 → g_rate
       - 语调模式 → g_pattern
       韵律指数 = 0.4*g_int_range + 0.3*g_rate + 0.3*g_pattern

     音质维度:
       - HNR → g_hnr
       - Jitter → g_jitter
       - Shimmer → g_shimmer
       音质指数 = 0.4*g_hnr + 0.3*g_jitter + 0.3*g_shimmer

  3. 综合评分
     综合性别评分 = 
       0.40 * 基频指数 +
       0.25 * 共振峰指数 +
       0.15 * 音色指数 +
       0.10 * 韵律指数 +
       0.10 * 音质指数

  4. 输出
     - 综合评分 (0-100)
     - 各维度分项评分
     - 典型分布区间归属百分比
```

**参考统计数据**（基于语音学文献）：
- 成年男性 F0：约 85-155 Hz（平均 ~120 Hz）
- 成年女性 F0：约 165-255 Hz（平均 ~210 Hz）
- F1/F2 性别差异：女性元音 F1 略高、F2 显著更高
- 语调范围：女性通常语调范围更大

## 四、关于"朗读指定文本"策略的讨论

### 4.1 什么是指定文本策略
市面上很多声音训练/分析系统要求用户朗读一段**预先设定好的标准文本**（如指定的句子、段落或特定韵母），然后基于这段标准化录音进行分析。

### 4.2 支持该策略的理由

| 优点 | 说明 |
|------|------|
| **可比性强** | 相同文本下，不同时间/不同人的录音更有对比意义 |
| **参数更稳定** | 特定音素（如元音 /a/ /i/ /u/）的共振峰更可靠 |
| **韵律分析更准** | 已知文本可以更好地分析语调、重音、节奏 |
| **训练目的明确** | 针对特定发音练习，便于追踪训练效果 |
| **减少变量** | 消除内容差异带来的参数波动 |

### 4.3 不支持该策略的理由

| 缺点 | 说明 |
|------|------|
| **使用门槛高** | 用户需要看到文本再朗读，多了一步操作 |
| **不自然** | 朗读文本的声音不如自然说话有代表性 |
| **场景有限** | 无法分析日常对话、演讲等真实场景 |
| **实现复杂** | 需要存储文本、展示文本、核对朗读完整性 |
| **语言限制** | 需要针对不同语言准备不同的标准文本 |

### 4.4 建议方案：双模式

考虑到两类需求都有价值，建议采用 **双模式设计**：

**模式一：自由录音模式（默认）**
- 用户发送任意语音即可分析
- 适合快速检测、日常记录、真实场景分析
- `/analyze` 和 `/expert` 都支持

**模式二：标准文本模式（可选）**
- 新增 `/standard` 命令，触发标准文本朗读流程
- Bot 发送一段标准文本（含特定元音/韵母/句子）
- 用户朗读后回复语音进行分析
- 该模式下的分析结果会打上 "standard" 标签
- 历史对比时，standard 模式的记录可以互相精准对比

**标准文本示例（中文）**：
```
请朗读以下内容（约 30-60 秒）：

元音延长：
啊——（保持5秒）
衣——（保持5秒）
乌——（保持5秒）

标准句：
你好，今天天气真好。
我正在练习我的声音。
希望声音能越来越好听。

数数字：
一二三四五六七八九十
```

### 4.5 实施建议
- **第一阶段**：先实现自由录音模式，快速上线核心功能
- **第二阶段**：根据用户反馈，再决定是否加入标准文本模式
- 即使加入标准文本模式，也**不强制**，保持选择权在用户手中

> **需要你确认**：是否在第一阶段就加入标准文本模式？还是先做自由模式，后续迭代再加？

## 六、技术实现要点

### 6.1 Termux 环境适配
```bash
# Termux 依赖安装
pkg update && pkg install -y python libportaudio fftw
pip install -r requirements.txt

# 环境变量配置
export TELEGRAM_BOT_TOKEN="your_token_here"
export DATA_DIR="/data/data/com.termux/files/home/voice_analyzer_data"
```

### 6.2 音频处理性能优化
- 参数提取在 `/analyze` 时全量完成，后续 `/expert` 直接读缓存
- 图表按需生成，生成后缓存到磁盘
- 使用 numpy 向量化计算提升速度
- Termux 环境下可考虑限制部分计算量大的参数精度

### 6.3 图表生成策略
- 后端使用 matplotlib (Agg 后端，无需显示设备)
- DPI: 120-150（平衡清晰度和文件大小）
- 暗色主题（深色背景 + 亮色曲线）
- 输出格式: PNG
- 每张图表附带简短文字说明

### 6.4 数据删除策略
- `/delete <id>` 删除单条记录及关联文件
- `/delete_all` 清空所有记录（需要用户二次确认）
- 删除时同时删除：音频文件、图表缓存、数据库记录
- 删除操作不可逆，Bot 会明确提示

## 八、风险评估与备选方案

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| Termux 性能有限，分析慢 | 用户等待时间长 | 限制时长 2 分钟，优化算法，缓存结果 |

| 存储空间不足 | 无法保存更多录音 | 提供删除功能，定期清理缓存图表 |
